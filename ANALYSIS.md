# ArmorStrike — Детальный разбор функционала проекта

Полный анализ исходного кода 3D-танкового экшена на **React 19 + TypeScript + Three.js + Vite + Tailwind**. Игра — одиночный режим «выживание волн»: игрок собирает танк в Гараже (корпус + башня) и сражается с волнами ИИ-ботов на процедурном заводском полигоне 150×150 с разрушаемыми укрытиями.

---

## 1. Технологический стек и точка входа

- **Сборка**: `vite` + `@vitejs/plugin-react`, плагин `vite-plugin-singlefile` (деплой в один HTML), Tailwind 4 (`@tailwindcss/vite`).
- **3D**: `three@0.185` (WebGL, PBR `MeshStandardMaterial`, тени, `PMREMGenerator`+`RoomEnvironment` для окружающего IBL-света, ACES tone-mapping).
- **UI**: `react@19` (StrictMode, `createRoot` в `main.tsx`), иконки `lucide-react`, утилиты `clsx`/`tailwind-merge`.
- **Звук**: процедурный `WebAudio` (без ассетов), динамический компрессор.
- **Стейт**: собственный класс `RunState` (в `localStorage`), без сторонних стейт-менеджеров.

Точка входа `main.tsx` → `App.tsx` (корневой layout, меню/гараж/HUD/экран поражения) → `Game` (всё ядро симуляции).

## 2. Архитектура: `Game` как ядро

Класс `Game` (`src/game/Game.ts:71`) — единый агрегатор. Он владеет рендерером, сценой, камерой (`CameraRig`), ареной (`Arena`), эффектами (`Effects`), пулом снарядов (`ProjectileManager`), вводом (`PlayerController`), звуком (`AudioFX`), состоянием забега (`RunState`) и менеджером волн (`WaveManager`).

**Режимы** `GameMode = 'menu' | 'garage' | 'playing' | 'over'` — состояние хранится в `RunState.mode`.

**Событийная шина**: `Game.addListener` / `emit(GameEvent)` — `GameEvent` (`Game.ts:54`) покрывает `playerHit` (с направлением удара), `enemyHit`, `kill`, `wave`, `shotFired`, `gameOver`, `pauseChanged`, `garageChanged`. Через неё React-HUD и `App` узнают о событиях боя.

**Единая фабрика оружия** `createWeapon()` (`Game.ts:228`) — единственная точка создания `RailgunWeapon`/`FlamethrowerWeapon`/`CannonWeapon` и для игрока, и для ботов (контракт `Weapon` из `weapons/types.ts`).

**Централизованная система урона** `damageSystem` (`Game.ts:88`): `applyDamage`, `applyKnockback`, `damageBlock` — все повреждения (танкам, блокам арены) идут через неё, что устраняет разветвления по `weaponType` в ядре.

## 3. Главный цикл и порядок обновления

`requestAnimationFrame` → `tick(ts)` (`Game.ts:565`):
- `dt = min(0.05, …)` — кламп дельты от просадок.
- Если `playing && !paused` → `step(dt)`.
- Иначе (пауза/меню) обновляются только мёртвые танки (`t.update`) для анимации гибели.
- Всегда: `arena.update`, `effects.update`, `cameraRig.update`, рендер кадра, синхронизация миникарты (`MinimapSystem.sync`).

`step(dt)` (`Game.ts:477`) — строгий порядок (важен для детерминизма):
1. Ввод игрока (`PlayerController.update` → `wantsFire` → `weapon.setFire`) + звук ручной перезарядки.
2. Обновление ИИ ботов (`AIController.update` для каждого из `waves.bots`) + `weapon.setFire` бота.
3. `WeaponSystem.update` — заряд/луч/частицы спец. оружий.
4. `TankSystem.update` — движение корпусов, физика, таймеры.
5. `TankFxSystem.update` — дым повреждений и пыль из-под гусениц.
6. `NameplateSystem.update` — таблички ботов.
7. `PhysicsSystem.resolveCollisions` — выталкивание из стен и разведение танков.
8. `projectiles.update` — полёт снарядов и все попадания.
9. `waves.update` — переход к следующей волне.
10. Смерть игрока → таймер 2 c → `mode='over'` + событие `gameOver`.
11. Реактивная струя нитро из кормы, звук двигателя, синхронизация миникарты.

**Смена режимов** (`Game.setMode`, `startRound`):

- Выход в меню/гараж (`Game.ts:254`): очистка боевых объектов, сброс паузы, показ 3D-предпросмотра.

## 6. Арена — завод 150×150

`Arena` (`src/game/Arena.ts`) строит процедурный/полу-статичный полигон «ЗАВОД-51». `ARENA = {size:150, wallH:7.5, wallT:3.5}` (`constants.ts:3`).

**Структура** (через модульные билдеры `arena/*.ts`):
- Пол (`factoryGroundTexture` — один 2048² canvas на всю арену: дороги, Ж/Д пути на западе, зоны погрузки, разливы масла, шины, люки, маркировки цехов), внешняя «пустота», стены ангара со световыми полосами и вывесками (`signTexture`).
- Зоны: центральный машинный зал (`centralHall` — с голографическим обелиском на крыше), козловой кран (`gantryCrane`, движущаяся тележка), литейный цех (`foundry`, пульсирующая «расплавленная» светимость), резервуарная группа (`silos`), контейнерный двор (`containerYard`), трубопроводная эстакада (`pipeRack`), трансформаторные блоки (`transformers`), рампы (`ramps`), рассеянный груз (`scattered` — ящики и бочки), дымовые трубы (`smokestacks` — эмиттеры стекового дыма), силуэты района за стенами (`skyline`), энергокупол и пыль (`atmosphere`).
- Все билдеры получают `ArenaBuildContext` (`arena/context.ts`) с хелперами `box()`, `addColliderBlock()`, регистраторами обелиска/крана/купола/дыма.

**Разрушаемые блоки**: `addColliderBlock(..., destructible:true, ...)` создаёт `BlockInfo` (HP, материалы, flash-состояние). `damageBlock(id, dmg)` (`Arena.ts:275`): уменьшает HP, при 0 — деактивирует коллайдер, удаляет меши и триггерит `onBlockDestroyed` (взрыв/дебрис). Флэш-подсветка при попадании (`b.flash`), вибрация при крит. уроне.

**Анимации арены** (`Arena.update`, `Arena.ts:290`): вращение купола/обелиска, мигание маяков (`beaconMats`), пульс свечения печи/расплава, качание тележки крана, стековый дым из труб (пулинг спрайтов, макс 44).

## 7. Танк — симуляция `TankEntity`

`TankEntity` (`src/game/Tank.ts:46`) — единая сущность для игрока и ботов.

**Физика корпуса** (`update`, `Tank.ts:143`):
- Скорость через `MathUtils.damp` к целевой (`throttle * maxFwd`, реверс `throttle * reverseSpeed`).
- Поворот: `yaw += steer * turnSpeed * agility * dt`, где `agility` растёт с относительной скоростью (манёвреннее на ходу).
- Нитро `boostActive` (`wantBoost = boosting && boostEnergy>min && throttle>0.15`) — множитель `BOOST.multiplier=1.5`, расход/восстановление энергии (`drainPerSec:0.5`/`rechargePerSec:0.28`).
- Позиция = скорость + вектор `knockback` (затухает `exp(-5.5 dt)`), `vel` пересчитывается для упреждения ИИ.
- **Саморемонт**: если `timeSinceHit > 5 c` и HP<max → +7 HP/с.
- **Башня**: плавное вращение `turretYaw` к `aimYaw - yaw` с ограничением шага `turretSpeed`.
- Анимация: `barrelKick` (откат ствола), прокрутка текстуры гусениц (`trackTex.offset.y`), `hitFlash` (эмиссия при попадании), пульс кольца под танком.
- Смерть: `deathT`, опускание ствола, затемнение материалов, скрытие кольца.

**Меш танка** (`tank/buildMesh.ts` + `hull.ts` + `turret.ts`):
- 3 корпуса: **Хантер** (сбалансированный), **Викинг** (низкий/быстрый), **Мамонт** (бронированный монолит, высокая башня). Высота башни зависит от корпуса.
- 3 башни: рельсотрон (ствол с `railGlowMat` — контролируемая эмиссия), огнемёт (две трубы + светящиеся кольца), пушка (ствол + кожух).
- Материалы: камуфляж (`camoTexture`), металл, светящиеся фары (`lampMat` = цвет команды), гусеницы (`trackTexture`). У игрока — антенна (`antenna:true`). Кольцо-ауры под танком (additive) цветом команды.

**Параметры** (`TankParams`): `maxHealth, speed, reverseSpeed, turnSpeed, turretSpeed, damage, shotCooldown, weaponType, range`.

## 9. Оружие — единый контракт `Weapon`

Интерфейс `Weapon` (`weapons/types.ts:42`): `setFire(active)`, `update(dt, ctx)`, `dispose()`, `getAmmoState()`. Контекст `WeaponDeps` (scene, effects, audio, damageSystem, projectiles, onShotFired). Состояние аммуниции делегируется оружию (`WeaponAmmoState`), что убирает ветвления в HUD/ядре.

### 9.1 Рельсотрон — `RailgunWeapon` (`RailgunWeapon.ts`)
Hitscan с FSM: `IDLE → CHARING → FIRING → COOLDOWN` (`RailgunState`).
- **Заряд**: старт по фронту нажатия (`setFire`), `chargeTime=1.1 c`, экспоненциальный рост свечения `railGlowMat` (`emissiveIdle 0.15 → emissiveCharged 4.5`), вибрация ствола.
- **Выстрел** (`executeFiring`): `Raycaster` (far=`range=120`) по меши танков (маппинг меш→танк через `parent`) + меши арены. **Сквозное пробитие**: урона падает ×`penetrationFactor=0.65` на каждую пробитую цель (`currentDamage *= 0.65`), стены/блоки останавливают луч. Блоку — `damageBlock`.
- **Луч**: растягиваемый `CylinderGeometry` (additive), затухание за `beamDuration=0.25 c`, точечные вспышки `muzzleLight`/`impactLight`.
- Откат `knockback=18`, `reloadTime=4.8 c`.

### 9.2 Огнемёт «Firebird» — `FlamethrowerWeapon` (`FlamethrowerWeapon.ts`)
Ближний бой, непрерывное удержание.
- **Частицы пламени**: пул `InstancedMesh` (160 частиц, отключён frustum-culling), спавн со скоростью 40/с, жизнь 0.55–0.9 c, градиент цвета (бело-жёлтый → оранжевый → красный → дым) через `setHSL`.
- **Overlap-check урона**: каждые `tickRate=0.1 c` цели в конусе `coneAngle=π/4` на дистанции `range=22` получают `damagePerTick=12` + лёгкий отброс.
- **Топливо**: `energyMax=100`, расход `28/с`, восстановление `22/с` (`getAmmoState` показывает запас). Звук `startFlameLoop`/`stopFlameLoop` (зацикленный шумовой lowpass).

### 9.3 Пушка «Смоки» — `CannonWeapon` (`CannonWeapon.ts`)
Скорострельная, мгновенно спавнит снаряд в `ProjectileManager`.
- `shotCooldown=0.28 c`, `magazine=10`, `fullReload=1.8 c`, `recoil`.
- Снаряд летит и обрабатывается в `Projectile.update` (фугасная площадь снаряда).

## 10. Снаряды — `ProjectileManager`

Пул из 42 снарядов (`Projectile.ts:42`), переиспользуемые меши (капсула + additive-спрайт свечения). `fire(owner, origin, dir, dmg, weaponType, range)` настраивает цвет/скорость/масштаб/параметры взрыва под тип.

`Projectile.update` (`Projectile.ts:135`): шаговая симуляция (`steps = ceil(speed/0.6)`) для туннель-фри детекции:
- **Коллизии со стенами/блоками**: для огнемёта — дым/искры; пушки — взрыв + `doSplash`; рельсотрона — вспышка. Блокирующие/деструктивные блоки получают `damageBlock`.
- **Попадание в танк** (`segmentHitsCircle`): урон + отброс по типу; для пушки — `doSplash` (радиус 5, урон = половина, убывает с дистанцией ×`falloff`, отброс от эпицентра).
- **Предел дальности**: взрыв в воздухе (пушка) или затухание (огонь/луч).
- **Светящийся след** (`trailPuff`) с разной частотой по типу.

## 12. Менеджер волн — `WaveManager`

`WaveManager` (`src/game/WaveManager.ts`):
- Точки спавна `SPAWN_POINTS` (7 по периметру), случайный неповторяющийся выбор.
- `spawnWave` (`WaveManager.ts:53`): `botsForWave(wave)=min(2+wave,5)` ботов. Статы `botStatsForWave` (`constants.ts:169`): `maxHealth=55+(w-1)*12`, `damage=10+w`, `speed=min(13.5,10.5+w*0.5)`, `turretSpeed`, `shotCooldown`, `sightRange=46`, `fireRange=40`, `aimError=max(0.05,0.14-w*0.012)`.
- Корпус/башня бота чередуются (`hulls[i%3]`, `turrets[(i+wave)%3]`), цвет из `COLORS.bots`. Прочность/урон масштабируются волной (`0.8+w*0.1` и `0.7+w*0.08`).
- Боты получают `randomPersona(wave)` — сложность растёт (реакция быстрее, ошибка прицела меньше).
- Переход: когда все боты мертвы → таймер `nextWaveT=2.2 c` (бонус за волну `SCORE.waveBonus` + гудок `waveHorn`), очистка мёртвых танков/табличек, спавн следующей волны.

## 13. Физика и геометрия — `physics.ts`

2D-AABB на плоскости XZ (`src/game/physics.ts`):
- `Collider` (`id, minX/maxX/minZ/maxZ, height, blocksShots, blocksSight, destructible, active, kind:'wall'|'block'|'ramp'`), фабрика `colliderFromCenter`.
- `resolveCircle(x,z,r,colliders)` — выталкивание круга из AABB (2 итерации, обработка центра внутри).
- `segmentHitsCollider` / `segmentHitT` — slab-метод пересечения сегмента с AABB (для LOS и камеры).
- `losClear(ax,az,bx,bz,…)` — прямая видимость (учитывает `blocksSight`).
- `segmentHitsCircle` — попадание сегмента в круг (для friendly-fire и снарядов).
- `clamp`, `wrapAngle` (нормализация угла).

`PhysicsSystem.resolveCollisions` (`systems/PhysicsSystem.ts`): выталкивание живых танков из коллайдеров (с потерей скорости при ударе) + попарное разведение танков (push-out по сумме радиусов).

## 14. Эффекты — `Effects`

`Effects` (`src/game/effects.ts`):
- **Искры**: `SparkPool` (800 частиц, `Points` + vertex colors, гравитация/drag, пулинг по курсору). Методы `burst` (взрывы), `jet` (струя нитро, отрицательная гравитация).
- **Дым**: пулинг спрайтов `spawnSmoke` (тёмный — от повреждений, светлый — пыль), макс 42.
- **Взрыв** `explosion`: световое ядро (`MeshBasicMaterial` additive, scale-up), ударная волна-кольцо по земле, искры, дым, `flashLight`, **обугливание** `addScorch` (текстура на полу, затухает).
- **Вспышки дула** `muzzle` (6 спрайтов-пула), **impact**-искры.
- **Тряска камеры** `addShake`/`getShake`: trauma-модель (интенсивность², затухание), мульти-синусоидальный шум + крен.
- **Амбиентная пыль** `ambient`: 140 точек дрейфуют вокруг центра (позиция игрока `setAmbientCenter`), обёртка в радиусе 95.
- `tankSmoke` (дым при HP<32%), `tankDust` (пыль при скорости>8).

## 15. Камера — `CameraRig`

`CameraRig` (`src/game/CameraRig.ts`):
- Режимы `menu` (орбита вокруг предпросмотра r=17, вращение танка), `garage` (r=8.5), `playing`, `over`.
- **Боевая камера**: третье лицо за спиной, управляется `input.camYaw`/`camPitch` (чувствительность `AIM_SENS_X/Y`, кламп тангажа `[-0.18,0.85]`). Дистанция 9.6, плавный `damp` (быстрее жив — `lam=14`, медленнее мёртв — `4`).
- **Обход препятствий**: `segmentHitT` против коллайдеров (h>2.5) — камера не уходит сквозь стену (поджимается к точке пересечения).
- **Динамический FOV**: расширяется со скоростью (+5) и нитро (+4).
- Тряска через `effects.getShake` (позиция + крен `camera.up`).

## 16. Звук — процедурный `AudioFX`

`AudioFX` (`src/game/audio.ts`): WebAudio без файлов.
- `ensure()` создаёт `AudioContext` + мастер-ган + **динамический компрессор** (защита от клиппинга при залпах).
- Примитивы: `osc` (осциллятор с ramp-частотой), `noise` (буфер белого шума + фильтр), `env` (огибающая).
- Звуки: заряд рельсотрона, петля огнемёта, выстрелы (square+noise для пушки/рельсотрона, bandpass для огня), взрыв, попадание по врагу/игроку, перезарядка, клик UI, гудок волны, смерть, **двигатель** (зацикленный sawtooth, частота/громкость зависят от скорости и буста).
- `muted` через `localStorage`-независимый флаг, мягкий `setTargetAtTime`.

## 17. HUD и UI (React)

`App.tsx` — корневой layout: canvas + overlay-эффекты (scanlines, vignette), маршрутизация по `uiMode`.
- **Главное меню**: 3D-предпросмотр (камера `menu`), карточка текущей сборки, кнопки ИГРАТЬ/ГАРАЖ, справка по управлению. ESC→пауза (в бою) / назад (в гараже); `M`→звук; `Enter`→старт.

- **`HUD.tsx`** (`src/components/HUD.tsx`): свой `requestAnimationFrame`-цикл читает `game.getHud()`; быстрые элементы через **refs** (броня, нитро, кольцо перезарядки conic-gradient), дискретные — через `forceUpdate`. Содержит:
  - **Прицел** (крест, hitmarker при попадании/килле, pulse при выстреле).
  - **Виньетка урона** + **направленный индикатор** (SVG-дуга, повёрнутая к атакующему по `playerHit.dir`).
  - **Миникарта** (canvas 172px): радарная развёртка (`createConicGradient`), статика (стены/рампы/блоки, блоки гаснут при разрушении через `MinimapSystem`), динамика (треугольники танков + линия башни).
  - Счёт/время (центр), лента убий (фид), счётчик целей (точки ботов), панель оружия (пипсы патронов/шкала топлива, индикатор ЗАРЯДКА/ПЕРЕЗАРЯДКА), броня+нитро (низ), подсказка управления (11 c), баннер волны, popup «+ ФРАГ», **табло боя** по `TAB` (имя/корпус/башня/оружие/броня).
- **`Garage.tsx`** (см. §8), **`PauseMenu.tsx`** (`src/components/PauseMenu.tsx`): статистика, сборка, кнопки ПРОДОЛЖИТЬ/НАЧАТЬ ЗАНОВО/В ГАРАЖ/В МЕНЮ/ЗВУК, подсказка ESC.

## 18. Системы — модульный слой

Тонкие stateless-модули `systems/*.ts` (единая точка обновления каждого аспекта):
- `TankSystem` — `for(t of tanks) t.update(dt)`.
- `WeaponSystem` — `for(t) t.weapon?.update(dt,{tanks,arena})`.
- `TankFxSystem` — дым повреждений (HP<32%) и пыль гусениц (скорость>8).
- `NameplateSystem` — позиция/видимость/HP табличек.
- `PhysicsSystem` — коллизии стен и танков.
- `MinimapSystem` — синхронизация `alive` блоков по `id` (O(n)).

## 19. Баланс по волнам

`constants.ts`: `botStatsForWave(wave)` и `botsForWave(wave)` масштабируют прочность, урон, скорость, скорость башни, кулдаун, **дальность обзора (46)**, **дальность огня (40)**, **ошибку прицела `aimError`↓** и **реакцию `react`↓** ботов. Персона `randomPersona(wave)` делает агрессию/упреждение/реакцию разнообразнее и точнее с волной. Игрок же фиксирован в выбранной сборке — сложность растёт чисто со стороны врага.

## Итог

ArmorStrike — модульный 3D-экшен, где `Game` агрегирует ECS-подобный конвейер систем (`AI → Weapon → Tank → Fx → Nameplate → Physics → Projectile → Waves`), единые контракты `Weapon`/`DamageSystem` устраняют разветвления по типу оружия, а весь визуал/звук генерируется процедурно (canvas-текстуры, шейдеры, WebAudio). Ключевые фишки: разрушаемая арена, 3×3 сборка танков, 3 принципиально разных оружия, FSM-ИИ с «характером» и упреждением, динамические волны, кастомная физика/камера/эффекты.

## Приложение: детальный разбор ИИ противников

> (Полный разбор из отдельного запроса — сохранён здесь для полноты документа.)

### Архитектура
ИИ вынесен в отдельный модуль `src/game/AI.ts`. Каждый бот управляется экземпляром `AIController`, который пишет в танк команды (`throttle`, `steer`, `aimYaw`, `boosting`), а флаг `wantsFire` читается оружейной системой. Главный цикл в `Game.ts:497` обновляет всех ботов и передаёт им контекст: игрок, список ботов, коллайдеры арены и границы.

### Состояния (конечный автомат)
У бота два состояния (`AI.ts:32`):
- **`patrol`** — патрулирование/разведка;
- **`engage`** — боевое преследование.

Переходы (`AI.ts:100-112`):
- Видит игрока (`canSee` = жив + дистанция < `sight` (46) + чистая линия огня `losClear`) → мгновенно в `engage` с задержкой реакции `reactT`.
- Не видит > 2.2 c → возврат в `patrol`, бот идёт проверить `lastSeen`.

### «Характер» бота — Persona
`AIPersona` (`AI.ts:16`) задаёт разнообразие:
- `aggro` (0..1) — дистанция боя и тяга к сближению;
- `react` — задержка реакции при обнаружении (уменьшается с волной);
- `lead` — упреждение по движущейся цели.

Генерится `randomPersona(wave)` (`AI.ts:22`), сложность растёт с волной.

### Механики движения (engage)
- **Удержание дистанции** под своё оружие: `prefRange()` (`AI.ts:63`) — огнемёт 7, рельсотрон 34+, пушка 20+. Бот сближается/отступает/маневрирует боком (`strafeDir`) в зависимости от `dist` vs `pref`.
- **Рулёжка**: ПИД-подобный поворот к точке назначения (`steer = clamp(diff*2.4, -1, 1)`), газ падает на поворотах.
- **Избегание препятствий** (`AI.ts:142`): зонд в 4.2 м впереди; если упёрся — выбирает свободную сторону (лево/право) на 0.6 c.
- **Антизастревание** (`AI.ts:166`): если едет, но `speed < 1` > 1.1 c — берёт новую точку.
- **Нитро** (`AI.ts:194`): агрессивные боты (`aggro > 0.5`) ускоряются при сближении.

### Механики башни и огня
- **Прицеливание** (`AI.ts:200`): для пушки — упреждение (`lead`) по вектору скорости игрока; для hitscan-оружия (рельсотрон/огнемёт) — прямо в цель. Добавляется случайная ошибка `aimError` (падает с волной).
- **Условия выстрела** (`AI.ts:213`): видимость + истекла реакция + дистанция < `fireRange` (40) + `fireTimer` готов + башня в допуске `aimTol` (огнемёт 0.3, рельсотрон 0.1, пушка 0.14).
- **Friendly fire** (`AI.ts:218`): не стреляет, если на линии танк-игрок есть другой бот (`segmentHitsCircle`).

### Патрулирование
Каждые 3-й манёвр бот идёт «разведать» окрестность игрока (`pickWaypoint`, `AI.ts:70`); доехал — крутит башней сканируя (`scanT`, `AI.ts:228`), затем новая точка.

### Баланс по волнам
`WaveManager.ts:89` и `constants.ts:169` масштабируют: `maxHealth`, `damage`, `aimError↓`, `react↓`, `sightRange` 46, `fireRange` 40 — боты становятся точнее и быстрее реагируют с каждой волной.

Коротко: ИИ — это FSM (патруль↔бой) с персонализированным «характером», удержанием дистанции под тип оружия, упреждением цели, избеганием препятствий, антизастреванием, нитро и защитой от friendly fire.
